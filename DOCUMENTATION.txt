# JobPilot AI — Full Project Documentation

**Version:** 1.0.0  
**Stack:** Node.js (Express) backend + React (Vite) frontend  
**Purpose:** AI-powered job application agent. Reads your CV, finds jobs, drafts outreach via Gmail/WhatsApp, and queues every message for human approval before sending.

## BACKEND

### `backend/src/index.js` — Server Entry Point

Creates the Express app, loads middleware, mounts all routes, and starts the server.

- `helmet` — security headers
- `cors` — allows frontend origin
- `express-rate-limit` — 500 req / 15 min
- JSON body limit: 15 MB
- Health check: `GET /api/health`
- In production: serves built frontend `dist/` as static files
- Port: `process.env.PORT` or `5000`

## BACKEND SERVICES

### `services/aiService.js` — Core AI Logic

All functions are pure JavaScript — no external AI API required in demo mode.

| Function | Description |
|---|---|
| `parseResumeText(text)` | Extracts name, email, phone, LinkedIn, GitHub, skills, education, experience from raw text. Calculates ATS score and improvement gaps. |
| `scoreJobMatch(profile, job)` | Token overlap scoring between resume and job. Returns `matchScore`, `atsScore`, `missingSkills`, `strengths`. |
| `analyzeJobRisk(job)` | Scans job text for scam signals (fee requests, personal emails, urgent language). Returns `riskLevel` (Low/Medium/High) and flags. |
| `researchCompany(job)` | Summarises company, hiring signals, tech stack, and recruiter questions from job data. |
| `tailorResumeForJob(profile, job)` | ATS-optimised headline, summary, and bullets. Shows ATS score before/after. |
| `generateInterviewPrep(profile, job)` | Elevator pitch, technical and behavioural questions, questions to ask interviewer. |
| `createFollowUpPlan(application, days)` | Generates a follow-up message due N days from now. |
| `generateDecisionReport(profile, job)` | Combines all above into one report. Final recommendation: Apply / Review / Skip. |
| `classifyInboxMessage({subject, body, from})` | Detects intent: offer / interview / rejection / follow_up_needed. |
| `draftOutreach({profile, job, channel})` | Drafts professional email (Gmail) or short WhatsApp message. |

---

### `services/resumeService.js`

| Function | Description |
|---|---|
| `extractText(buffer, mimetype, name)` | PDF via `pdf-parse`, DOCX via `mammoth`, or plain text |
| `parseResumeUpload(file)` | Validates file, extracts text, calls `parseResumeText()` |

| Function | Description |

### `services/whatsappService.js`

| Function | Description |

| `sendViaMeta(to, body)` | Sends via Meta Cloud API (graph.facebook.com) |

---

### `services/jobService.js`

| Function | Description |
|---|---|
| `normalizeJob(job)` | Fills missing fields with defaults |
| `searchJobs({store, query, location, type, minSalary, experience})` | Filters jobs; respects user blacklist |
| `scoreJobsForResume(jobs, resume)` | Scores + sorts all jobs by match score |

---

### `services/cryptoService.js`

| Function | Description |
|---|---|
| `encryptJson(value)` | AES-256-GCM encrypt any JSON → `{iv, tag, data}` |
| `decryptJson(payload)` | Decrypts `{iv, tag, data}` back to JSON |

Used to safely store Gmail OAuth tokens in `store.json`.

---

### `services/sendPolicy.js`

| Function | Description |
|---|---|
| `assertApprovedApplication(app)` | Throws 403 if not in `approved` status |
| `assertDailyLimit(store, channel)` | Throws 429 if daily send count exceeds `dailySendLimit` (default 15) |
| `incrementDailyUsage(store, channel)` | Increments today's send counter |

### `db/store.js` — File-Based Database

Uses `data/store.json` as the single database file.

| Function | Description |
|---|---|
| `ensureStore()` | Creates `store.json` with defaults if missing |
| `readStore()` | Parses JSON; fills missing keys |
| `writeStore(store)` | Writes formatted JSON to disk |
| `updateStore(mutator)` | Read → mutate → write atomically |
| `addAuditLog(action, details)` | Appends audit entry (max 250 kept) |
| `publicSummary(store)` | Returns dashboard stats |

**Store shape:**
```json
{
  "users": [],
  "resumes": [],
  "jobs": [],
  "applications": [],
  "messages": [],
  "followUps": [],
  "inboxEvents": [],
  "auditLogs": [],
  "integrations": { "gmail": {}, "whatsapp": {} },
  "dailyUsage": {}
}
```

Provides `demoUser` and `seedJobs` to populate a fresh store with sample data.

### `Landing.jsx`
Marketing page. Shows app description, 4 capability cards, product preview mockup. Links to resume upload and job feed.

Command center. Shows 4 metric cards, review queue with Approve/Send buttons, latest resume summary.

## Data Flow Summary

```
1. Upload Resume
   ResumeManager → POST /api/resume/parse
   → resumeService.extractText() → aiService.parseResumeText()
   → saved to store.json resumes[]

2. Search Jobs
   JobFeed → GET /api/jobs/search
   → jobService.searchJobs() + scoreJobsForResume()
   → returned with match scores and risk levels

3. Queue Applications
   JobFeed → POST /api/applications/queue
   → scoreJobMatch + generateDecisionReport + draftOutreach + createFollowUpPlan
   → saved to store.json applications[] with status: pending_review

4. Approve Application
   Dashboard/Applications → POST /api/applications/:id/approve
   → status: approved, snapshot saved

5. Send Application
   → POST /api/gmail/send OR /api/whatsapp/send
   → sendPolicy: approval check + daily limit check
   → gmailService or whatsappService sends message
   → status: applied (or sent_demo in demo mode)
   → follow-up plan auto-created

6. Inbox Classification
   InboxMonitor → POST /api/inbox/classify
   → aiService.classifyInboxMessage()
   → matching application status auto-updated
     (interview / offer / rejected / follow_up_needed)
```
