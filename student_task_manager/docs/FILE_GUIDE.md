# File-by-File Guide

## Root files

- `pubspec.yaml` — project name, Dart range, Supabase/formatting dependencies, and test tools.
- `analysis_options.yaml` — beginner-friendly static analysis rules.
- `.env.example` — names of the configuration values; it contains no real key.
- `.gitignore` — prevents generated output, IDE settings, and private local files from entering Git.
- `README.md` — portfolio overview, setup, commands, architecture, and documentation links.
- `.github/workflows/flutter_ci.yml` — runs formatting, analysis, and tests on GitHub pushes and pull requests.

## Application entry and configuration

- `lib/main.dart` — starts Flutter, checks configuration, initializes Supabase, and launches the app.
- `lib/app.dart` — defines the Material application, authentication gate, and missing-setup screen.
- `lib/config/supabase_config.dart` — reads the Supabase URL and public anonymous key from `--dart-define`.

## Core shared code

- `lib/core/constants/app_constants.dart` — app name and form length limits in one place.
- `lib/core/theme/app_theme.dart` — teal/orange Material 3 theme with accessible contrast and consistent form styles.
- `lib/core/utils/error_message.dart` — turns Supabase exceptions into understandable messages.

## Data and services

- `lib/models/task.dart` — task fields, priority/status enums, conversion from database rows, and conversion back to maps.
- `lib/services/auth_service.dart` — signup, login, current-user, and logout calls.
- `lib/services/task_service.dart` — realtime task reading plus create, update, delete, and status operations.

## Screens

- `lib/screens/auth/login_screen.dart` — validated login form and link to signup.
- `lib/screens/auth/signup_screen.dart` — validated registration form with password confirmation.
- `lib/screens/dashboard/dashboard_screen.dart` — task stream, status counts, filtering, error/empty states, and CRUD actions.
- `lib/screens/tasks/task_form_screen.dart` — reusable add/edit form for all required task fields.

## Widgets

- `lib/widgets/task_card.dart` — displays one task with priority, status, due date, completion checkbox, edit, and delete actions.
- `lib/widgets/status_summary_card.dart` — small reusable count card used by the responsive dashboard.
- `lib/widgets/empty_task_state.dart` — helpful message and action when a filter has no tasks.

## Backend and tests

- `supabase/schema.sql` — users/tasks tables, foreign keys, checks, indexes, triggers, RLS policies, and realtime setup.
- `test/models/task_test.dart` — unit tests for task conversion logic.
- `test/widgets/status_summary_card_test.dart` — widget test for a dashboard component.

## Documentation

- `docs/SRS.md` — purpose, scope, environment, constraints, interfaces, and security.
- `docs/REQUIREMENTS.md` — numbered functional and non-functional requirements.
- `docs/DATABASE_DESIGN.md` — tables, relationship, normalization, indexes, and RLS.
- `docs/DIAGRAMS.md` — Mermaid use case, ER, and class diagrams with plain-language explanations.
- `docs/TEST_CASES.md` — manual acceptance cases and automated-test notes.

## Generated platform folders

`android`, `ios`, `web`, `windows`, `linux`, and `macos` are generated by `flutter create .`. They contain platform launchers and should normally not contain business logic. This environment did not have Flutter installed, so generate them using the README command before the first run.
